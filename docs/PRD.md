# 盒记 — PRD

> 个人盒饭档口记账工具 · v1.0 · 2026-06-09

---

## 1. 产品概述

**一句话定位**：为单兵作战的盒饭档口老板设计的轻量记账 App，专注"今晚接单 → 明天配送 → 晚上对账"的每日闭环。

**平台**：Android（侧载 APK 个人使用，不上应用市场）

**技术栈**：uni-app (Vue 3) + Vite + TypeScript + Pinia + plus.sqlite（本地 SQLite）

---

## 2. 产品名称

**暂定名：盒记**（HeJi）

- 命名思路：`盒`（盒饭）+ `记`（记账），音同"合计"，有"统统一笔算清楚"的语感
- 备选：`餐账` / `盒账` / `老板记账`
- 最终名称待定，由产品负责人确认

---

## 3. 目标用户与使用场景

### 3.1 用户画像

- **角色**：自己接单 + 自己配送的小型盒饭档口老板
- **业务规模**：每日 20-100 单，覆盖周围 3 公里
- **支付方式**：微信群接龙 + 私信转账为主，偶有现金
- **技术能力**：会用微信、会装 APK，不需要花哨的引导

### 3.2 业务节奏

| 时间 | 动作 | 工具 |
|---|---|---|
| 20:00-22:00 | 微信群发菜单 → 收私信订餐 → 收转账 | 微信（App 外） |
| 次日 09:00-10:30 | 整理订餐 → 备菜做饭 | 微信（App 外） |
| 次日 11:30 / 17:30 | 配送（3 公里内，电动车） | 微信（App 外） |
| 次日 21:00-22:00 | 对账、补录支出、看月卡余量 | **盒记** |

盒记的职责 = 替换手写小本本 + 微信记账小程序，**不替代微信群接单流程**。

## 4. 核心功能

### F1 · 订单管理
- 录入订单：选客户、餐次（午餐/晚餐）、金额、支付方式（微信/现金/月卡扣次）、备注
- 订单列表：按日期筛选；显示当日所有订餐
- 订单状态：待配送 / 已配送 / 已取消
- 配送时一键切换状态

### F2 · 客户与月卡
- 客户列表：姓名、微信、电话、类型（普通/月卡）、备注
- 月卡管理：开卡（总餐数、起止日期、金额）、查看使用进度（已用/剩余）
- 月卡到期前 N 天提醒
- 月卡客户下单时自动扣次、单笔订单记 0 收入（实际收入在开卡时记）

### F3 · 支出管理
- 录入支出：日期、分类、金额、备注
- 默认支出分类（可自定义增删）：
  - 菜品（食材进货）
  - 工具（锅碗瓢盆、设备维修）
  - 耗材（餐盒、筷子、塑料袋）
  - 配送（电费、车辆维护）
  - 其他

### F4 · 统计与对账
- **日维度**：今日收入、支出、利润、订单数
- **周维度**：本周每日趋势、对比上周
- **月维度**：本月利润、各分类支出占比
- **时间筛选**：任意日/周/月自由切换
- **客户维度**：单个客户月消费、点餐频次

### F5 · 仪表盘（首页）
- 今日概览：订单数 / 收入 / 支出 / 利润（一屏四数）
- 今日订餐明细（快速看谁订了什么）
- 即将到期月卡提醒
- 今日待配送订单数（醒目标记）

### F6 · 备份与恢复
- 一键导出：全量数据 → JSON 文件，保存到本地或通过微信"文件传输助手"传到电脑
- 一键导入：选 JSON 文件 → 覆盖恢复
- 后续扩展：CSV 单表导出、局域网 HTTP 同步

## 5. 产品使用路径

### 5.1 主路径：每日对账闭环

```
20:00  [微信群] 发明天的菜单（App 外）
   ↓
20:00-22:00  [微信] 客户私信订餐 + 转账（App 外）
   ↓
22:00-22:30  [盒记] 批量录入订单
   ├─ 已有客户 → 选客户 → 录金额 → 标支付方式
   ├─ 新客户 → 先加客户 → 再录订单
   └─ 月卡客户 → 自动扣次（订单金额 0）
   ↓
次日 11:00 / 17:00  [盒记] 配送时把订单标"已配送"
   ↓
次日 21:00  [盒记] 对账
   ├─ 看 Dashboard：今日利润
   ├─ 补录支出：买菜 / 餐盒 / 耗材
   └─ 看月卡：处理到期提醒
   ↓
每周日晚  [盒记] 周复盘（统计页）
   ↓
每月 1 号  [盒记] 月复盘 + 导出 JSON 备份
```

### 5.2 次要路径

| 触发 | 路径 |
|---|---|
| 临时买耗材 | 任何时候 → 支出页 → 快速录一笔 |
| 新客户加微信 | 客户页 → 新建 → 填基础信息 |
| 客户开月卡 | 客户详情 → 开月卡 → 填总餐数 + 金额 + 起止日期 |
| 换手机 / 重装 | 设置页 → 导入 JSON → 全量恢复 |
| 每周保险 | 设置页 → 导出 JSON → 微信发到"文件传输助手"存电脑 |

## 6. 非目标（v1 不做）

明确划清边界，避免范围蔓延：

- ❌ **多用户登录 / 实时云同步**（走导入导出方案，零账号体系）
- ❌ **在线支付集成**（微信转账 App 外完成）
- ❌ **菜单发布 / 微信群管理**（仍在微信群内）
- ❌ **配送路径规划**（3 公里内老板自行安排）
- ❌ **食材库存追踪**（进货按支出记一笔，不追踪剩余）
- ❌ **PDF 报表导出**（v1 用 JSON，后续再考虑）
- ❌ **iOS / 小程序 / H5 版本**（v1 只做 Android APK）
- ❌ **数据图表复杂可视化**（v1 数字 + 简单条形图够用）

## 8. 验收标准（v1.0 简版）

满足以下条件视为 v1.0 达标：

- [ ] 能在真机侧载安装并打开
- [ ] 录入一笔订单 → Dashboard 当日订单数 +1、收入更新
- [ ] 录入一笔支出 → Dashboard 当日支出更新、利润 = 收入 - 支出
- [ ] 切换日/周/月统计 → 数字与手算一致
- [ ] 开一张月卡 → 客户详情显示进度
- [ ] 月卡客户下单 → 自动扣次、单笔记 0 收入
- [ ] 导出 JSON → 卸载重装 → 导入 JSON → 数据完整恢复
