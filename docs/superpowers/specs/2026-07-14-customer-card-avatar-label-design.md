# 客户列表次卡身份头像标识设计

## 目标

`src/pages/me/customers/list.vue` 不再在头像区显示客户姓名首字，而是按客户当前次卡状态显示身份标识：

- 客户当前存在至少一张仍有剩余次数的 active 次卡时，显示“次”。
- 没有上述次卡时，显示“普”。
- 已用完的次卡不计入次卡用户判定。

## 数据口径

次卡用户条件为：

```sql
status = 'active' AND used_meals < total_meals
```

这个条件明确以“当前可用余额”为准，不依赖客户是否曾经开过卡。

## 实现方案

1. 在 `src/api/meal-cards.ts` 增加批量查询方法，一次返回所有当前次卡用户的 `customer_id`，避免为每位客户分别查询次卡。
2. 客户列表页刷新时，同时刷新客户列表和次卡用户 ID 集合。
3. 模板根据客户 ID 是否在集合中，在现有头像区显示“次”或“普”。
4. 保留现有搜索、拼音分组、右侧索引、折扣角标和详情跳转行为。

## 变更边界

- 不修改 SQLite schema、迁移或备份格式。
- 不把次卡状态写入 `Customer` 领域类型或 Pinia 客户 store。
- 不改动客户详情、订单客户选择器或其他页面的头像展示。

## 异常处理

客户或次卡身份查询失败时，沿用页面现有的“客户加载失败”提示，不使用过期的身份集合冒充当前数据。

## 验收标准

- 拥有 active 且剩余次数大于 0 的客户显示“次”。
- 从未开卡或所有次卡均已用完的客户显示“普”。
- 客户开新卡、用完次卡或校正总次数后，重新进入客户列表可看到最新标识。
- 客户列表原有搜索、分组、索引跳转和详情跳转不受影响。
- `pnpm type-check` 和 `pnpm lint` 通过。
